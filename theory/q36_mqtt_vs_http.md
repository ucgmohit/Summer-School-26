# Q36: MQTT vs HTTP Comparison

| Feature | MQTT | HTTP |
|--------|------|------|
| Architecture | Publish–Subscribe | Request–Response |
| Data Model | Lightweight messages | Heavy request/response |
| Power | Low | High |
| Latency | Very low | Higher |
| IoT Suitability | Excellent | Limited |
| Security | TLS, broker auth | HTTPS |

## Conclusion
MQTT is best for large IoT deployments like 1000-node smart agriculture because it is lightweight, scalable, and energy efficient.
